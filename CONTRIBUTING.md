## How to contribute to rcmdr or rcmdrmisc

#### **Did you find a bug?**

* **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/RCmdr-Project/rcmdr/issues).

* If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/RCmdr-Project/rcmdr/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample** current and the expected behavior that is not occurring.

#### **Did you write a patch that fixes a bug?**

* Open a new GitHub pull request with the patch.

* For rcmdr only request on devel branch will be accepted

* Ensure the description clearly describes the problem and solution.

#### **Do you intend to add a new feature or change an existing one?**

* Suggest your change in the [rcmdr discussions](https://github.com/RCmdr-Project/rcmdr/discussions) and start writing code.

* Do not open an issue on GitHub until you have collected positive feedback about the change. GitHub issues are primarily intended for bug reports and fixes.
